SBMLsqueezer 1.3

This page summarizes important information for working with SBMLsqueezer version 1.3. Note that this is not the most recent version, and it is strongly recommend to download a newer version of this program.

Publications

SBMLsqueezer: a CellDesigner plug-in to generate kinetic rate equations for biochemical networks
Andreas Dräger, Nadine Hassis, Jochen Supper, Adrian Schröder, and Andreas Zell
BMC Systems Biology 2008, 2:39 (30 April 2008).
[BibTeX], [pdf]

Introduction

Modeling biochemical networks has gained much attention with the raise of the young research branch systems biology. The creation of kinetic equations, so-called rate laws, constitutes one crucial step in the model building pipeline that leads to many errors due to its difficulty. One of the most important software programs in systems biology is CellDesigner [KLH+07, KFM+05]. This program allows the user to create SBML process diagrams of biological networks graphically. However, there is currently no method to derive kinetic equations from the Systems Biology Graphical Notation (SBGN), used to represent the network. Here we present SBMLsqueezer [Has+07, DHS+08], a plug-in for the latest CellDesigner version that generates rate equations for biochemical networks based on the the context of each reaction. Thereby, SBMLsqueezer takes the number and kind of reacting species, the type of modifiers such as inhibitors, catalysts, or activators, and the type of reaction into account. In this way, a selection of applicable kinetic equations is generated from which the user can choose the desired rate law. User defined settings specify which equation to apply for any type of reaction. Furthermore, all equations can be created one by one using a context menu associated to each reaction. All equations are written as MathML directly into the SBML [HFS+03] file. An export function saves the resulting ordinary differential equation system as LaTeX or text file for further processing. Since version 1.3, SBMLsqueezer also runs in a stand-alone mode allowing to embed the rate law generation into more complex automatic tasks.

For further information and a complete documentation of all currently supported kinetic equations we refer to the publications and supplementary material.

Currently supported rate laws

Rate laws for metabolic processes

  • (Generalized) mass-action rate law with numerous orders [HS1996]
  • Uni-uni Michaelis-Menten kinetics [HS1996]
  • Irreversible non-modulated non-interacting reactant enzymes (SBO)
  • Bi-uni enzyme mechanisms [Seg1993, Bis2000, Cor2004]
    • Random-order mechanism
    • Ordered mechanism
  • Bi-bi enzyme reactions [Seg1993, Bis2000, Cor2004]
    • Random-order mechanism
    • Ordered mechanism
    • Ping-pong mechanism
  • Modular rate laws for enzymati reactions [LUK2010]
    • Power-law modular rate law (PM)
    • Common modular rate law (CM)
    • Direct binding modular rate law (DM)
    • Simultaneous binding modular rate law (SM)
    • Force-dependent modular rate law (FM)
  • Convenience kinetics [LK07]
    • Thermodynamically dependent form
    • Thermodynamically independent form
  • (Generalized) Hill equation [Cor2004]

Rate laws for gene-regulatory processes

  • Hill-Hinze equation [HHL+07]
  • Hill-Radde equation [RK2007]
  • Linear additive network models
    • General form
    • NetGenerator form [TGD+07]
  • Non-linear additive network models
    • General form
    • NetGenerator form [TGD+07]
    • Vohradský equation [VV2007]
    • Weaver equation [WWS1999]
  • S-systems [Sav1969]
  • H-systems [SHS+06]

How to get started?

After you have opened an existing or created a new SBML model in SBMLsqueezer or CellDesigner, SBMLsqueezer provides two major ways to create kinetic rate laws for a given SBML file in CellDesigner:

  1. Generation of all equations in one go
  2. All equations can be generated one by one as well.
You can also upload your model to the SBMLsqueezer online version that provides the same functionality but does not require the installation of any software on your local machine. Section Quicktour of this web page as well as the Tutorial provide more detailed explanations of the usage of SBMLsqueezer.

Example Networks

Two example files are available from this web page for testing, studying and illustration purposes:

  • An artificial model of a cytosol
  • A model of the T-cell signalling cascade starting at the stimulation of CD3/CD28 and ending with the translocation of the nuclear factor of activated T-cells.

Installation Instructions

For detailed explanations how to install, maintain and use SBMLsqueezer, the Tutorial and the Kinetic Law Supplement are very helpful as these documents provide a comprehensive introduction and description of the program. Section Quick Tour of this web page provides a quick introduction into the usage of SBMLsqueezer. To integrate SBMLsqueezer as an equation generating core into your application, the API documentation gives the necessary information.

  • Requirements:
    • Minimal computer: CPU 800 MHz, 128 MB RAM, 4 MB free disk space.
    • Java VM 1.5 or later (available from Sun),
    • For the plug-in mode: CellDesigner 4.0α or higher (available from CellDesigner)
  • Download SBMLsqueezer from the download section of this web page
  • If you want to use SBMLsqueezer as a plug-in for CellDesigner, please procede as follows:
    • Install CellDesigner (see the CellDesigner web site for further instructions)
    • Copy the SBMLsqueezer jar-file into the plugin directory of CellDesigner. For instance
      /opt/CellDesigner4.1beta/plugin/under Linux,
      /Applications/CellDesigner4.1beta/plugin/under Mac OS X, or
      C:\Programs\CellDesigner4.1beta\plugin\under Windows,
    Please not that this works for all SBMLsqueezer versions.
  • For the stand-alone mode (requires at least SBMLsqueezer version 1.3), you can copy the SBMLsqueezer JAR file to any directory in your system. You can start the application by double clicking on the JAR file. It might be necessary to install libSBML version 4.0.0 before using SBMLsqueezer in stand-alone mode. If it does not run after a double click, please set the path for libSBML for your respective system. To this end, please see the instructions on the libSBML homepage. It might be necessary to start SBMLsqueezer from the console, e.g., by typing:
    java -cp SBMLsqueezer.1.3.jar:<path to libSBML> org.sbml.squeezer.SBMLsqueezer
    A list of all currently available command-line options can be found "below. The example start script run.sh may demonstrate how to configure your system appropriately.

Example start script for SBMLsqueezer 1.3

Here you can download the example start script run.sh, which demonstrates how to set up the necessary environment variables for a Linux operating system. For your particular platform it may be necessary to modify this script, but it may serve as a good start point and ease invoking SBMLsqueezer 1.3 in its stand-alone mode.  

Overview of all command-line options for SBMLsqueezer 1.3

This page gives an overview about all available command-line options in SBMLsqueezer version 1.3. If the argument requires the values true or false, the value can be omitted to set the flag to true, for instance, --gui and --gui=true lead to the same effect (to show the graphical user interface). The key-value pairs must not contain any white spaces. The separator between key and value is always the symbol =. All command-line options start with -- and correspond to a configuration key that can be used within the Application Programming Interface (API). The API keys and the command-line options differ in their case, but actually SBMLsqueezer does ignore the case when evaluating command-line options. But underscores are not allowed in the API keys. The following overview contains possible values and a short description of each option.
--gui GUI
true or false Can be used in combination with = true or = false or just --gui.
--sbml-file SBML_FILE
valid path to a valid SBML file SBML input file
--sbml-out-file SBML_OUT_FILE
valid path to a valid SBML file Specifies the file where SBMLsqueezer writes its SBML output.
--open-dir OPEN_DIR
valid path to an arbitrary directory Standard directory where SBML files can be found.
--save-dir SAVE_DIR
valid path to an arbitrary directory Standard directory where SBML or Text files can be saved.
--show-sbml-warnings SHOW_SBML_WARNINGS
true or false Can be true or false, depending on if the user wants to see SBML warnings.
--check-for-updates CHECK_FOR_UPDATES
true or false Decide whether or not SBMLsqueezer should search for updates at start-up.
--kinetics-uni-uni-type KINETICS_UNI_UNI_TYPE
class name of a kinetic law that implements InterfaceUniUniKinetics This argument determines the default rate law to be applied for uni-uni type enzyme reactions
--kinetics-bi-uni-type KINETICS_BI_UNI_TYPE
class name of a kinetic law that implements InterfaceBiUniKinetics Specifies the default rate law for bi-uni type enzyme reactions.
--kinetics-bi-bi-type KINETICS_BI_BI_TYPE
class name of a kinetic law that implements InterfaceBiBiKinetics The default rate law for bi-bi type reactions
--kinetics-none-enzyme-reactions KINETICS_NONE_ENZYME_REACTIONS
class name of a kinetic law that implements InterfaceNonEnzymeKinetics Determins the default kinetic law to be applied for reactions that are catalyzed by non-enzymes or that are not catalyzed at all.
--kinetics-gene-regulation KINETICS_GENE_REGULATION
class name of a kinetic law that implements InterfaceGeneRegulatoryKinetics Determins the key for the standard kinetic law to be applied for reactions that are identified to belong to gene-regulatory processes, such as transcription or translation.
--kinetics-other-enzyme-reactions KINETICS_OTHER_ENZYME_REACTIONS
class name of a kinetic law that implements InterfaceArbitraryEnzymeKinetics Determins the key for the standard kinetic law to be applied for reactions that are identified to be enzyme-catalyzed (with or without explicit catalyst) and that do not belong to one of the other standard enzyme-catalysis schemes.
--type-standard-version TYPE_STANDARD_VERSION
0 (for cat), 1 (for hal), 2 (for weg) One of the following values: cat, hal or weg (important for Liebermeister's standard kinetics).
--type-unit-consistency TYPE_UNIT_CONSISTENCY
0 (to bring species to substance units), 1 (to bring species to concentration units) How to ensure unit consistency in kinetic equations? One way is to set each participating species to an initial amount and to set the unit to mmole. The other way is to set the initial concentration of each species, set the unit to mmole per l and to multiply the species with the size of the surrounding compartment whenever it occurs in a kinetic equation. Hence, this type paramter belongs to two values.
--opt-all-reactions-are-enzyme-catalyzed OPT_ALL_REACTIONS_ARE_ENZYME_CATALYZED
true or fals If true, all reactions within the network are considered enzyme reactions. If false, an explicit enzymatic catalyst must be assigned to a reaction to obtain this status.
--opt-generate-kinetic-law-for-each-reaction OPT_GENERATE_KINETIC_LAW_FOR_EACH_REACTION
true or false If true a new rate law will be created for each reaction irrespective of whether there is already a rate law assigned to this reaction or not.
--opt-add-new-parameters-always-globally OPT_ADD_NEW_PARAMETERS_ALWAYS_GLOBALLY
true or false If true all parameters are stored globally for the whole model (default) else parameters are stored locally for the respective kinetic equationthey belong to.
--opt-treat-all-reactions-reversible OPT_TREAT_ALL_REACTIONS_REVERSIBLE
true or false Property that decides whether to set all reactions to reversible before creating new kinetic equations.
--opt-max-number-of-reactants OPT_MAX_NUMBER_OF_REACTANTS
any non-negative integer The maximal number of reactants so that the reaction is still considered plausible.
--opt-warnings-for-too-many-reactants OPT_WARNINGS_FOR_TOO_MANY_REACTANTS
true or false
--opt-default-compartment-initial-size OPT_DEFAULT_COMPARTMENT_INITIAL_SIZE
any non-negative real value If not specified the value corresponding to this argument will be used to initialize the size of compartments.
--opt-default-species-initial-value OPT_DEFAULT_SPECIES_INITIAL_VALUE
any non-negative real value If not specified the value corresponding to this argument will be used to initialize species depending on their hasOnlySubstanceUnits property as initial amount or initial concentration.
--opt-default-value-of-new-parameters OPT_DEFAULT_VALUE_OF_NEW_PARAMETERS
any non-negative real value The value that is set for newly created parameters.
--opt-set-boundary-condition-for-genes OPT_SET_BOUNDARY_CONDITION_FOR_GENES
true or false Decide whether or not to set the boundary condition for genes to true.
--opt-remove-unnecessary-parameters-and-units OPT_REMOVE_UNNECESSARY_PARAMETERS_AND_UNITS
true or false If true parameters and units that are never referenced by any element of the model are deleted when creating kinetic equations with SBMLsqueezer.
--opt-ignore-these-species-when-creating-laws OPT_IGNORE_THESE_SPECIES_WHEN_CREATING_LAWS
a komma-separated list without white spaces of valid MIRIAM-compliant compound identifiers Allows the user to ignore species that are annotated with the given compound identifiers when creating rate laws for reactions that involve these species. For instance, water or single protons can often be ignored when creating rate equations, hence simplifying the resulting rate equations. Preselected are the KEGG compound identifiers for water and protons
--possible-enzyme-generic POSSIBLE_ENZYME_GENERIC
true or false Determins whether or not generic proteins are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-rna POSSIBLE_ENZYME_RNA
true or false Determins whether or not RNA molecules are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-complex POSSIBLE_ENZYME_COMPLEX
true or false Determins whether or not enzyme complexes are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-truncated POSSIBLE_ENZYME_TRUNCATED
true or false Determins whether or not trunkated proteins are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-receptor POSSIBLE_ENZYME_RECEPTOR
true or false Determins whether or not receptors are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-unknown POSSIBLE_ENZYME_UNKNOWN
true or false Determins whether or not unknown molecules are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-antisense-rna POSSIBLE_ENZYME_ANTISENSE_RNA
true or false Determins whether or not antisense RNA molecules are accepted as enzymes when catalyzing a reaction.
--possible-enzyme-simple-molecule POSSIBLE_ENZYME_SIMPLE_MOLECULE
true or false Determins whether or not simple molecules are accepted as enzymes when catalyzing a reaction.
--latex-dir LATEX_DIR
a valid path to an arbitrary directory Standard directory where LaTeX files can be stored.
--latex-paper-size LATEX_PAPER_SIZE
letter, legal, executive, or a0..a10, b0..b10, c0..c10, d0..d10 The paper size for LaTeX documents.
--latex-font-size LATEX_FONT_SIZE
8, 9, 10, 11, 12, 14, 17 The font size for LaTeX documents.
--latex-ids-in-typewriter-font LATEX_IDS_IN_TYPEWRITER_FONT
true or false Key that decides whether or not identifiers should be written in typewriter font when these occur in mathematical equations.
--latex-landscape LATEX_LANDSCAPE
true or false Decides whether to set the LaTeX document in landscape or portrait mode.
--latex-title-page LATEX_TITLE_PAGE
true or false Decides whether to create a separate title page instead of a simple heading.
--latex-names-in-equations LATEX_NAMES_IN_EQUATIONS
true or false Decides whether to write the names or the identifiers of NamedSBase object in equations.
