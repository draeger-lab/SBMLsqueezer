* SBMLsqueezer: Update of the current year and adaptation to the latest version of Garuda

* SubmodelController: Avoids a null pointer exception in case that for some reason no kinetics has been created for a reaction within the submodel.

* Made the dialogs for the kinetics selection and also for SABIO-RK resizable.

* SBMLsqueezerUI: Adaptation to latest version of Garuda; also shifted the garuda menu entry to the edit menu to make it less prominent; fixed a bug that after loading kinetics from SABIO-RK the SBML tree was not updated.

* SABIORKWizard: This dialog is now placed relative to the main window and has become resizable, which is both important!

* JDialogWizard: Since in SBMLsqueezer the user selects already all or just one reaction in a model, the dialog has been changed so that these are pre-selected. The user can, however, still go back and change this selection. The default size of the dialog has been reduced with respect to small displays, where the dialog might otherwise not even pop up. It is also now resizable.

* Improved the build script.

* General: Updated old-style <code>…</code> to the new {@code …} and code style cleaning; in SABIO-RK part, tables don't have to fill a panel, Auto-resize has been switched of in order to make columns arbitrarily large. At the moment, this should be improved because sometimes columns are too little.
